# Vantage 本地测试指南（给测试同事）

在你自己电脑上跑一遍：自己启动后端 + 装插件 + 采集 + 看数据。全程本地，不影响别人。

## 需要
- 一台 Mac（已安装 **Node.js**；没有的话去 https://nodejs.org 装 LTS 版）
- Claude Code

---

## 第 1 步：启动后端（收数据的小服务）

解压收到的 `vantage` 文件夹。打开**终端**，执行：

```bash
cd <你解压的路径>/vantage/server
npm install      # 第一次要装依赖，等它跑完
npm start        # 启动后端，保持这个终端窗口开着别关
```

看到 `后端已启动 http://localhost:3000` 就成功了。**这个窗口要一直开着**（关了后端就停了）。

---

## 第 2 步：装插件（在 Claude Code 里输入）

```
/plugin marketplace add <你解压的路径>/vantage
/plugin install vantage@dgcrane
```

装完**重启一次 Claude Code**，然后：

```
/vantage:setup
```

按提示填 **姓名**（部门按公司通讯录自动填；不在册才需手选部门）。看到"初始化完成"即可。

---

## 第 3 步：用一下，让它采集

- **Claude Code**：新开一个窗口，随便聊两句（比如"帮我写个 hello world"），然后 `exit` 退出。
- **Codex**（如果你用）：不用任何操作、也不用 `/hooks` 信任。`/vantage:setup` 已装好后台定时任务，
  **每小时**自动扫一次 Codex 会话并采集。想立刻看到结果，终端手动跑一次即可：
  `node ~/.vantage/agent/reconcile.cjs --only codex`

---

## 第 4 步：看采到的数据

终端里执行：

```bash
curl -s -H "Authorization: Bearer dev-token-change-me" http://localhost:3000/stats
```

能看到你的姓名、用了哪个工具、模型、token 数就说明成功了。

也可以直接用文本编辑器打开数据文件看每条记录：
```
vantage/server/data/usage.jsonl
```

---

## 卸载（测完想清理）

```
/plugin uninstall vantage@dgcrane
```
再删 Codex 定时任务和本机目录：
```bash
launchctl bootout gui/$(id -u)/com.dgcrane.vantage.codex 2>/dev/null   # mac
rm -rf ~/.vantage
```
后端在终端按 `Ctrl + C` 停掉即可。

---

## 遇到问题

- 后端 `npm start` 报错 → 多半是没装 Node，或没先 `npm install`
- `/plugin marketplace add` 报错 → 路径写错了，用解压后 `vantage` 文件夹的完整路径
- 采不到数据 → 确认后端终端还开着、`/vantage:setup` 跑过、并且是**装插件之后新开**的会话（只采安装之后的）
